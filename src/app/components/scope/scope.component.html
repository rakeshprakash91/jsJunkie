Scope

Scope of a variable is defined as the portion of code where in we can access the variable. In javascript we have 2 scopes.

  • Function scope
  • Global scope
Function scope A variable defined inside a function would be accessible across the function, irrespective of the place where it is defined(thanks to variable hoisting). Consider a variable defined inside a for loop, it would be accessible across the function where-in it is defined. 
      function abc(){{'{'}}
       console.log('before for loop: ',i);
       for(var i = 0; i < 5; i++){{'{'}}
            console.log('inside for loop: ',i);
       {{'}'}}
       console.log('after for loop: ',i);
      {{'}'}}

abc(); //would log
before for loop: undefined
inside for loop: 0
inside for loop: 1
inside for loop: 2
inside for loop: 3
inside for loop: 4
after for loop: 5
As we can see, there won't be an error thrown. The reason being the declaration is available at the top of the function/its current scope. Since no initialization has been done, the first console.log gave undefined. Within the for loop, it logged 0 to 4, and post the for loop 5, since at the end of the for loop the value of i is 5.

Global scope As the name suggests, a variable declared globally would be accessible across all the functions in the script. Do not use global variable declarations unless required. The reason being, globally declared variables can be accessed & modified by functions/methods across the script. 
      (function(){{'{'}}
        var x = 5;
        function abc (){{'{'}}
            console.log(x); // 5
            x++;
        {{'}'}}
        abc();
        console.log(x); // 6
       {{"}"}})()

In the above code we have enclosed the whole code inside an IIFE to execute the entire code. On invoking the method abc, it logs 5 in the console, since variable x is defined globally. But within the method abc, the value of x is incremented, since it is globally declared variable, the value gets modified. Which is evident by the following console.log output.
This however can be avoided if we have a local/function scope variable. Consider the following code snippet 
        (function(){{'{'}}
        var x = 5;
        function abc (){{'{'}}
            var x = 5;
            console.log(x); // 5
            x++;
            console.log(x); // 6
        {{'}'}}
        abc();
        console.log(x); // 5
       {{"}"}})()

The reason why the global variable value is not affected is because, within the function we had a variable, whose scope was within the function. So whatever modifications were done, was limited to the function itself.

Note

While declaring global variables assign a namespace, so it doesn't clash with other global variable with same name

var myGlobalVars = {{'{'}} gVar1 : 10, gVar2 : 20 {{'}'}}

In this case we can be sure of preventing global variable clash & accidental modification of value. To access the global variables in the above format we use
myGlobalVars.gVar1
